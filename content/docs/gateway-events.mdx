# Gateway Events

The real-time gateway delivers events to connected clients over a ConnectRPC server-streaming RPC. Events cover everything from new messages to presence updates to moderation actions.

## Connecting

### Protocol

The gateway uses **ConnectRPC** (HTTP/2 + Protocol Buffers) for efficient binary streaming. Connect to it by calling the `Identify` RPC on the `GatewayService`:

```typescript
import { createClient } from "@connectrpc/connect";
import { createConnectTransport } from "@connectrpc/connect-node";
import { GatewayService } from "gateway/generated";

const transport = createConnectTransport({
	baseUrl: "https://msggw.oliqor.lol",
});
const client = createClient(GatewayService, transport);

const stream = client.identify({
	token: "<clerk_jwt>",
	intents: 0b111111111111111, // all intents
	largeThreshold: 50,
	shard: [0, 1], // [shard_id, shard_count]
});

for await (const event of stream) {
	const seq = event.sequence;
	switch (event.event.case) {
		case "messageCreate":
			console.log("New message:", event.event.value);
			break;
		case "presenceUpdate":
			console.log("Presence:", event.event.value);
			break;
	}
}
```

### Identify Parameters

| Field             | Type    | Required | Description                                                 |
| ----------------- | ------- | -------- | ----------------------------------------------------------- |
| `token`           | string  | Yes      | Clerk JWT or bot token                                      |
| `intents`         | int32   | No       | Bitfield of events to receive (default: all non-privileged) |
| `large_threshold` | int32   | No       | Member count threshold for "large" guilds (default: 50)     |
| `shard`           | int32[] | No       | `[shard_id, shard_count]` for sharded connections           |

### Ready Event

The first event after a successful identify is `READY`, which includes:

- Your user info
- List of guilds with channels, members, and roles
- Session ID for resuming
- Gateway protocol version

## Intents

Intents control which events you receive. They are a bitfield — combine them with OR (`|`).

| Intent                      | Bit       | Value     | Events Controlled                                        |
| --------------------------- | --------- | --------- | -------------------------------------------------------- |
| `GUILDS`                    | `1 << 0`  | `1`       | Guild create/update/delete, channel/role/thread events   |
| `GUILD_MEMBERS`             | `1 << 1`  | `2`       | Member add/remove/update, members chunk (**privileged**) |
| `GUILD_BANS`                | `1 << 2`  | `4`       | Ban add/remove                                           |
| `GUILD_EMOJIS_AND_STICKERS` | `1 << 3`  | `8`       | Emoji and sticker updates                                |
| `GUILD_INTEGRATIONS`        | `1 << 4`  | `16`      | Integration updates                                      |
| `GUILD_WEBHOOKS`            | `1 << 5`  | `32`      | Webhook updates                                          |
| `GUILD_INVITES`             | `1 << 6`  | `64`      | Invite create/delete                                     |
| `GUILD_VOICE_STATES`        | `1 << 7`  | `128`     | Voice state/server updates, voice ready                  |
| `GUILD_PRESENCES`           | `1 << 8`  | `256`     | Presence updates (**privileged**)                        |
| `GUILD_MESSAGES`            | `1 << 9`  | `512`     | Message create/update/delete, bulk delete                |
| `GUILD_MESSAGE_REACTIONS`   | `1 << 10` | `1024`    | Reaction add/remove/remove all                           |
| `GUILD_MESSAGE_TYPING`      | `1 << 11` | `2048`    | Typing start                                             |
| `DIRECT_MESSAGES`           | `1 << 12` | `4096`    | DM message create/update/delete                          |
| `DIRECT_MESSAGE_REACTIONS`  | `1 << 13` | `8192`    | DM reaction add/remove                                   |
| `DIRECT_MESSAGE_TYPING`     | `1 << 14` | `16384`   | DM typing start                                          |
| `MESSAGE_CONTENT`           | `1 << 15` | `32768`   | Access to message content (**privileged**)               |
| `GUILD_SCHEDULED_EVENTS`    | `1 << 16` | `65536`   | Scheduled event create/update/delete/user add/remove     |
| `AUTOMOD_CONFIGURATION`     | `1 << 20` | `1048576` | Auto-mod rule create/update/delete                       |
| `AUTOMOD_EXECUTION`         | `1 << 21` | `2097152` | Auto-mod action execution                                |

### Privileged Intents

Three intents are **privileged** and may require additional verification for large bots:

- `GUILD_MEMBERS` — member join/leave/update events
- `GUILD_PRESENCES` — online/offline/idle status changes
- `MESSAGE_CONTENT` — access to message text content

### Events Without Intent Requirements

These events are always delivered regardless of intents:

- `READY`, `RESUMED`
- `USER_UPDATE`
- `MESSAGE_ACK`

## Event Reference

### Guild Events

| Event          | Intent   | Description                                          |
| -------------- | -------- | ---------------------------------------------------- |
| `GUILD_CREATE` | `GUILDS` | You joined a guild, or guild data is sent on connect |
| `GUILD_UPDATE` | `GUILDS` | Guild properties were changed                        |
| `GUILD_DELETE` | `GUILDS` | You were removed from a guild, or it was deleted     |

### Channel Events

| Event                 | Intent   | Description                      |
| --------------------- | -------- | -------------------------------- |
| `CHANNEL_CREATE`      | `GUILDS` | A channel was created            |
| `CHANNEL_UPDATE`      | `GUILDS` | A channel was updated            |
| `CHANNEL_DELETE`      | `GUILDS` | A channel was deleted            |
| `CHANNEL_PINS_UPDATE` | `GUILDS` | A message was pinned or unpinned |

### Thread Events

| Event                   | Intent   | Description                                      |
| ----------------------- | -------- | ------------------------------------------------ |
| `THREAD_CREATE`         | `GUILDS` | A thread was created                             |
| `THREAD_UPDATE`         | `GUILDS` | A thread was updated (archived, locked, renamed) |
| `THREAD_DELETE`         | `GUILDS` | A thread was deleted                             |
| `THREAD_MEMBER_UPDATE`  | `GUILDS` | Your thread membership was updated               |
| `THREAD_MEMBERS_UPDATE` | `GUILDS` | Members were added or removed from a thread      |

### Message Events

| Event                 | Intent           | Description                            |
| --------------------- | ---------------- | -------------------------------------- |
| `MESSAGE_CREATE`      | `GUILD_MESSAGES` | A message was sent                     |
| `MESSAGE_UPDATE`      | `GUILD_MESSAGES` | A message was edited                   |
| `MESSAGE_DELETE`      | `GUILD_MESSAGES` | A message was deleted                  |
| `MESSAGE_DELETE_BULK` | `GUILD_MESSAGES` | Multiple messages were deleted at once |

### Reaction Events

| Event                         | Intent                    | Description                |
| ----------------------------- | ------------------------- | -------------------------- |
| `MESSAGE_REACTION_ADD`        | `GUILD_MESSAGE_REACTIONS` | A reaction was added       |
| `MESSAGE_REACTION_REMOVE`     | `GUILD_MESSAGE_REACTIONS` | A reaction was removed     |
| `MESSAGE_REACTION_REMOVE_ALL` | `GUILD_MESSAGE_REACTIONS` | All reactions were removed |

### Member Events

| Event                 | Intent          | Description                           |
| --------------------- | --------------- | ------------------------------------- |
| `GUILD_MEMBER_ADD`    | `GUILD_MEMBERS` | A user joined the guild               |
| `GUILD_MEMBER_REMOVE` | `GUILD_MEMBERS` | A user left or was kicked/banned      |
| `GUILD_MEMBER_UPDATE` | `GUILD_MEMBERS` | A member's roles or nickname changed  |
| `GUILD_MEMBERS_CHUNK` | —               | Response to `RequestGuildMembers` RPC |

### Role Events

### Presence Events

### Voice Events

### DM Events

| Event                | Intent                     | Description               |
| -------------------- | -------------------------- | ------------------------- |
| `DM_MESSAGE_CREATE`  | `DIRECT_MESSAGES`          | DM message was sent       |
| `DM_MESSAGE_UPDATE`  | `DIRECT_MESSAGES`          | DM message was edited     |
| `DM_MESSAGE_DELETE`  | `DIRECT_MESSAGES`          | DM message was deleted    |
| `DM_REACTION_ADD`    | `DIRECT_MESSAGE_REACTIONS` | Reaction added in DM      |
| `DM_REACTION_REMOVE` | `DIRECT_MESSAGE_REACTIONS` | Reaction removed in DM    |
| `DM_TYPING_START`    | `DIRECT_MESSAGE_TYPING`    | User started typing in DM |

### User Events

| Event         | Intent          | Description                            |
| ------------- | --------------- | -------------------------------------- |
| `USER_UPDATE` | — (always sent) | Your user profile was updated          |
| `MESSAGE_ACK` | — (always sent) | You acknowledged messages in a channel |

### Interaction Events

| Event                | Intent                 | Description                                            |
| -------------------- | ---------------------- | ------------------------------------------------------ |
| `INTERACTION_CREATE` | — (always sent)        | An interaction was created (command, component, modal) |
| `TYPING_START`       | `GUILD_MESSAGE_TYPING` | A user started typing                                  |

## Other Gateway RPCs

Besides `Identify`, the gateway service supports:

| RPC                   | Description                                                                     |
| --------------------- | ------------------------------------------------------------------------------- |
| `Resume`              | Resume a disconnected session using a session ID and last sequence number       |
| `Heartbeat`           | Keep the connection alive (send periodically)                                   |
| `RequestGuildMembers` | Request the full member list for a guild (returns `GUILD_MEMBERS_CHUNK` events) |

## Messaging RPCs

The `MessagingService` provides RPCs for common operations without REST:

| RPC                    | Description                       |
| ---------------------- | --------------------------------- |
| `SendMessage`          | Send a message to a channel       |
| `EditMessage`          | Edit a message                    |
| `DeleteMessage`        | Delete a message                  |
| `GetMessages`          | Retrieve messages from a channel  |
| `AddReaction`          | Add a reaction to a message       |
| `RemoveReaction`       | Remove a reaction                 |
| `PinMessage`           | Pin a message                     |
| `UnpinMessage`         | Unpin a message                   |
| `AckMessage`           | Acknowledge messages in a channel |
| `UpdatePresence`       | Update your presence status       |
| `UpdateActivity`       | Set your current activity         |
| `StartTyping`          | Broadcast a typing indicator      |
| `RespondToInteraction` | Respond to an interaction         |
