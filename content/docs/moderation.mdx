# Moderation

Oliqor provides a layered moderation system: manual tools for moderators (bans, kicks), automated content filtering (auto-mod rules), and a complete audit trail of all administrative actions.

---

## Bans

Bans prevent a user from rejoining the guild. Unlike kicks, banned users cannot use invite links until the ban is removed.

### List Guild Bans

```
GET /v1/guilds/{guildId}/bans
```

Requires `BAN_MEMBERS` permission.

**Query parameters**:

| Parameter | Type   | Default | Description                    |
| --------- | ------ | ------- | ------------------------------ |
| `limit`   | number | 100     | Max bans to return             |
| `after`   | string | —       | Return bans after this user ID |

**Response 200**:

```json
[
	{
		"reason": "Repeated harassment",
		"user": {
			"id": "user_def456",
			"username": "baduser",
			"global_name": null,
			"avatar": null,
			"bot": false,
			"system": false,
			"flags": 0,
			"public_flags": 0
		}
	}
]
```

---

### Get Guild Ban

```
GET /v1/guilds/{guildId}/bans/{userId}
```

Returns a specific ban entry. Returns 404 if the user is not banned.

---

### Create Guild Ban

```
PUT /v1/guilds/{guildId}/bans/{userId}
```

Requires `BAN_MEMBERS` permission.

**Request body** (optional):

| Field               | Type   | Default | Description                                           |
| ------------------- | ------ | ------- | ----------------------------------------------------- |
| `reason`            | string | null    | Reason for the ban (shown in audit log)               |
| `deleteMessageDays` | number | 0       | Delete the user's messages from the last N days (0–7) |

**What happens when a user is banned**:

1. A ban record is created.
2. The user's guild membership is removed.
3. The user is disconnected from any voice channels.
4. A `GUILD_BAN_ADD` gateway event is fired.
5. A `GUILD_MEMBER_REMOVE` gateway event is fired.
6. An audit log entry is created with type `MEMBER_BAN_ADD`.

---

### Remove Guild Ban (Unban)

```
DELETE /v1/guilds/{guildId}/bans/{userId}
```

Requires `BAN_MEMBERS` permission. The user is not automatically re-added to the guild — they need a new invite.

Fires a `GUILD_BAN_REMOVE` gateway event.

## Auto-Moderation

Auto-moderation rules automatically detect and act on content that violates guild policies.

### Auto-Mod Rule Object

```json
{
	"id": "rule_abc",
	"guild_id": "guild_abc",
	"name": "Block bad words",
	"creator_id": "user_abc123",
	"event_type": "MESSAGE_SEND",
	"trigger_type": "KEYWORD",
	"trigger_metadata": {
		"keyword_filter": ["badword", "offensive*"]
	},
	"actions": [
		{ "type": "BLOCK_MESSAGE" },
		{
			"type": "SEND_ALERT_MESSAGE",
			"metadata": { "channelId": "ch_mod_alerts" }
		}
	],
	"enabled": true,
	"exempt_roles": ["role_mod", "role_admin"],
	"exempt_channels": ["ch_bot_commands"],
	"createdAt": "2024-01-15T10:00:00.000Z",
	"updatedAt": "2024-01-15T10:00:00.000Z"
}
```

---

### List Auto-Mod Rules

```
GET /v1/guilds/{guildId}/automod/rules
```

Requires `MANAGE_COMMUNITY` permission.

**Query parameters**:

| Parameter | Type   | Description                    |
| --------- | ------ | ------------------------------ |
| `limit`   | number | Max rules to return            |
| `after`   | string | Return rules after this ID     |

---

### Get Auto-Mod Rule

```
GET /v1/guilds/{guildId}/automod/rules/{ruleId}
```

Requires `MANAGE_COMMUNITY` permission.

---

### Create Auto-Mod Rule

```
POST /v1/guilds/{guildId}/automod/rules
```

Requires `MANAGE_COMMUNITY` permission.

**Request body**:

| Field             | Type   | Required | Description                                   |
| ----------------- | ------ | -------- | --------------------------------------------- |
| `name`            | string | Yes      | Rule name                                     |
| `eventType`       | string | Yes      | Event that triggers the rule (e.g. `MESSAGE_SEND`) |
| `triggerType`     | string | Yes      | Trigger type (e.g. `KEYWORD`, `SPAM`)         |
| `triggerMetadata` | object | No       | Trigger configuration                         |
| `actions`         | array  | Yes      | Actions to take when rule fires               |
| `enabled`         | boolean| No       | Whether the rule is active (default `true`)   |
| `exemptRoles`     | array  | No       | Role IDs to exempt from this rule             |
| `exemptChannels`  | array  | No       | Channel IDs to exempt from this rule          |

Fires an `AUTO_MODERATION_RULE_CREATE` gateway event.

---

### Update Auto-Mod Rule

```
PATCH /v1/guilds/{guildId}/automod/rules/{ruleId}
```

Requires `MANAGE_COMMUNITY` permission. All fields optional.

Fires an `AUTO_MODERATION_RULE_UPDATE` gateway event.

---

### Delete Auto-Mod Rule

```
DELETE /v1/guilds/{guildId}/automod/rules/{ruleId}
```

Requires `MANAGE_COMMUNITY` permission.

Fires an `AUTO_MODERATION_RULE_DELETE` gateway event.

---

### Auto-Mod Gateway Events

| Event                              | Description                                     |
| ---------------------------------- | ----------------------------------------------- |
| `AUTO_MODERATION_RULE_CREATE`      | A rule was created                              |
| `AUTO_MODERATION_RULE_UPDATE`      | A rule was updated                              |
| `AUTO_MODERATION_RULE_DELETE`      | A rule was deleted                              |
| `AUTO_MODERATION_ACTION_EXECUTION` | A rule was triggered and an action was executed |

## Audit Log

The audit log is an immutable, append-only record of all administrative actions in a guild.

### Get Guild Audit Log

```
GET /v1/guilds/{guildId}/audit-logs
```

Requires `VIEW_AUDIT_LOG` permission.

**Query parameters**:

| Parameter | Type   | Default | Description                                   |
| --------- | ------ | ------- | --------------------------------------------- |
| `userId`  | string | —       | Filter by the user who performed the action   |
| `type`    | string | —       | Filter by event type (e.g., `MEMBER_BAN_ADD`) |
| `before`  | string | —       | Return entries before this audit log entry ID |
| `limit`   | number | 50      | Max entries to return (1–100)                 |

**Response 200**:

```json
{
	"audit_log_entries": [
		{
			"id": "entry_abc",
			"user_id": "user_abc123",
			"target_id": "user_def456",
			"action_type": "MEMBER_BAN_ADD",
			"changes": [
				{
					"key": "reason",
					"new_value": "Spamming"
				}
			],
			"reason": "Spamming",
			"created_at": "2024-01-15T10:00:00.000Z"
		}
	]
}
```

### Audit Log Event Types

#### Guild

| Event Type     | Description                 |
| -------------- | --------------------------- |
| `GUILD_UPDATE` | Guild settings were changed |

#### Channels

| Event Type                 | Description                      |
| -------------------------- | -------------------------------- |
| `CHANNEL_CREATE`           | Channel was created              |

| `CHANNEL_DELETE`           | Channel was deleted              |

| `CHANNEL_OVERWRITE_DELETE` | Permission overwrite was removed |
