# Bots & Interactions

Bots (applications) extend Oliqor with custom functionality. They can respond to slash commands, send messages, react to events, and manage guild resources.

## Concepts

### Applications

An **application** is the bot registration itself. It has a name, description, owner, and a secret token used for authentication.

### Commands

**Commands** are the slash commands that users invoke (e.g., `/ping`, `/ban @user`). Commands can be registered globally (available in all guilds) or per-guild.

### Interactions

An **interaction** is the event that fires when a user invokes a command, clicks a button, or submits a modal. The gateway dispatches interactions to the bot's webhook URL.

### Bot Installation

Bots must be **installed** in a guild before their commands are available there. Installation is done by a guild admin.

---

## Bot Authentication

Bot endpoints use a static `Bot-Token` header:

```
Bot-Token: <64-char hex token>
```

The token is shown **once** at application creation. If lost, generate a new one with the reset endpoint (this invalidates the old token).

## Applications

### Create Application

```
POST /v1/applications
```

**Request body**:

| Field            | Type   | Required | Constraints   | Description                          |
| ---------------- | ------ | -------- | ------------- | ------------------------------------ |
| `name`           | string | Yes      | 1–100 chars   | Bot display name                     |
| `description`    | string | No       | Max 500 chars | Bot description                      |
| `interactionUrl` | string | No       | Valid URL     | Webhook URL for interaction dispatch |

**Response 200**:

```json
{
	"id": "bot_abc",
	"name": "My Bot",
	"description": "A helpful bot",
	"interaction_url": "https://mybot.example.com/interactions",
	"owner_id": "user_abc123",
	"token": "<64-char hex token>",
	"created_at": "2024-01-15T14:30:00.000Z"
}
```

> **Save the token immediately.** It is returned only once and cannot be retrieved again. Use `POST /v1/applications/{botId}/reset-token` to generate a new one if lost.

### Get Current Application

```
GET /v1/applications/@me
```

Returns the authenticated bot's full application info. Requires `Bot-Token` header.

**Response 200** includes `install_count` and a list of registered `commands`.

---

### Update Application

```
PATCH /v1/applications/{botId}
```

Owner only.

**Request body** (all optional):

| Field            | Type    | Description                          |
| ---------------- | ------- | ------------------------------------ |
| `name`           | string  | Bot display name                     |
| `description`    | string  | Bot description                      |
| `avatarUrl`      | string  | Bot avatar URL                       |
| `interactionUrl` | string  | Webhook URL for interaction dispatch |
| `isPublic`       | boolean | Whether the bot is publicly listed   |

---

### Reset Bot Token

```
POST /v1/applications/{botId}/reset-token
```

Owner only. **Immediately invalidates** the old token.

**Response 200**:

```json
{
	"token": "<new 64-char hex token>"
}
```

---

### Delete Application

```
DELETE /v1/applications/{botId}
```

Owner only. Permanently deletes the bot, all its commands, and all installations.

## Commands

Commands define the slash commands users can invoke. They can be registered globally or scoped to a specific guild.

### Command Object

```json
{
	"id": "cmd_abc",
	"name": "ping",
	"description": "Replies with pong",
	"type": 1,
	"options": {}
}
```

| `type` | Name       | Description                   |
| ------ | ---------- | ----------------------------- |
| `1`    | CHAT_INPUT | Slash command (e.g., `/ping`) |
| `2`    | USER       | Context menu on a user        |
| `3`    | MESSAGE    | Context menu on a message     |

---

### Global Commands (Bot-Token auth)

#### Bulk Overwrite Global Commands

```
PUT /v1/applications/{botId}/commands
```

Replaces **all** global commands. Recommended for syncing on bot startup.

```json
[
	{
		"name": "ping",
		"description": "Replies with pong",
		"type": 1
	}
]
```

#### Create / List / Delete Global Commands

```
POST   /v1/applications/{botId}/commands
GET    /v1/applications/{botId}/commands
DELETE /v1/applications/{botId}/commands/{commandId}
```

---

### Guild Commands (Bot-Token auth)

Guild commands are only available in a specific guild. The bot must be installed there.

```
PUT    /v1/applications/{botId}/guilds/{guildId}/commands      — Bulk overwrite
POST   /v1/applications/{botId}/guilds/{guildId}/commands      — Create
GET    /v1/applications/{botId}/guilds/{guildId}/commands/{id} — Get
PATCH  /v1/applications/{botId}/guilds/{guildId}/commands/{id} — Update
DELETE /v1/applications/{botId}/guilds/{guildId}/commands/{id} — Delete
```

---

### Command Permissions (Bot-Token auth)

Override which roles/users/channels can use specific commands:

```
GET /v1/applications/{botId}/guilds/{guildId}/commands/permissions
PUT /v1/applications/{botId}/guilds/{guildId}/commands/permissions
GET /v1/applications/{botId}/guilds/{guildId}/commands/{commandId}/permissions
PUT /v1/applications/{botId}/guilds/{guildId}/commands/{commandId}/permissions
```

## Bot Installation

### Install Bot in Guild

```
POST /v1/guilds/{guildId}/bots/{botId}
```

Requires `MANAGE_COMMUNITY` permission.

**Request body**:

| Field         | Type   | Required | Description              |
| ------------- | ------ | -------- | ------------------------ |
| `permissions` | string | No       | Permission bitfield      |

**Response 200**:

```json
{
	"bot_id": "bot_abc",
	"guild_id": "guild_abc",
	"permissions": "256"
}
```

### Uninstall Bot

```
DELETE /v1/guilds/{guildId}/bots/{botId}
```

### List Installed Bots

```
GET /v1/guilds/{guildId}/bots
```

**Query parameters**: `limit`, `after`

**Response 200** — array of:

```json
{
	"bot_id": "bot_abc",
	"name": "My Bot",
	"description": "A helpful bot",
	"avatar_url": null,
	"permissions": "256",
	"installed_at": "2024-01-15T14:30:00.000Z"
}
```

## Interactions

### The Interaction Lifecycle

```
1. User invokes /ping in a guild channel
2. Client sends POST /v1/interactions with command details
3. Gateway looks up the command from installed bots
4. Gateway POSTs the interaction to the bot's interactionUrl webhook
5. Bot responds synchronously with one of:
   a. Immediate message (type 4)
   b. Deferred response (type 5) — "bot is thinking..."
   c. Modal dialog (type 9)
6. For deferred responses, bot later calls POST /v1/webhooks/{id}/{token}
```

### Invoke Slash Command

```
POST /v1/interactions
```

```json
{
	"commandName": "ping",
	"channelId": "ch_general",
	"communityId": "guild_abc",
	"options": [{ "name": "target", "value": "alice" }]
}
```

**Response 200**:

```json
{
	"id": "int_abc",
	"type": 2,
	"token": "<interaction_token>",
	"data": {},
	"modal": null
}
```

---

### Component Interaction

```
POST /v1/interactions/component
```

Fired when a user clicks a button or selects from a dropdown.

```json
{
	"customId": "confirm_button",
	"messageId": "msg_123",
	"channelId": "ch_general",
	"communityId": "guild_abc",
	"componentType": 2,
	"values": []
}
```

| Component Type | Name               |
| -------------- | ------------------ |
| `1`            | Action Row         |
| `2`            | Button             |
| `3`            | String Select      |
| `4`            | Text Input         |
| `5`            | User Select        |
| `6`            | Role Select        |
| `7`            | Mentionable Select |
| `8`            | Channel Select     |

## Webhook Callbacks (Bot-Token auth)

After a deferred response, use these endpoints to send the actual reply.

### Send Follow-up Message

```
POST /v1/webhooks/{webhookId}/{webhookToken}
```

```json
{
	"content": "Here is the result!",
	"embeds": [],
	"components": [],
	"flags": 0
}
```

### Edit Original Response

```
PATCH /v1/webhooks/{webhookId}/{webhookToken}/messages/@original
```

### Delete Original Response

```
DELETE /v1/webhooks/{webhookId}/{webhookToken}/messages/@original
```

## Bot Webhook Payload

When the gateway dispatches an interaction to your bot's `interactionUrl`, it sends a POST with this structure:

```json
{
	"id": "int_abc",
	"type": 2,
	"token": "<interaction_token>",
	"application_id": "bot_abc",
	"channel_id": "ch_general",
	"guild_id": "guild_abc",
	"member": {
		"user": { "id": "user_abc123", "username": "alice" },
		"roles": ["role_mod"],
		"permissions": "104324673"
	},
	"data": {
		"name": "ping",
		"options": []
	}
}
```

Your bot should respond with a JSON body indicating the response type:

```json
{
	"type": 4,
	"data": {
		"content": "Pong!"
	}
}
```

| Response Type                        | Value | Description                               |
| ------------------------------------ | ----- | ----------------------------------------- |
| CHANNEL_MESSAGE_WITH_SOURCE          | `4`   | Immediate message response                |
| DEFERRED_CHANNEL_MESSAGE_WITH_SOURCE | `5`   | Acknowledge, respond later                |
| DEFERRED_UPDATE_MESSAGE              | `6`   | Acknowledge component, no visible loading |
| UPDATE_MESSAGE                       | `7`   | Edit the original message (component)     |
| AUTOCOMPLETE_RESULT                  | `8`   | Return autocomplete suggestions           |
| MODAL                                | `9`   | Open a modal dialog                       |

## OAuth for Bots

Bots can support OAuth2 to allow users to authorize them via the standard flow.

### Enable OAuth

```
POST /v1/applications/{botId}/oauth
```

Enables OAuth 2.0 for a bot application. Returns `client_id` and `client_secret` — the secret is shown only once.

**Request body**:

| Field          | Type  | Required | Description                     |
| -------------- | ----- | -------- | ------------------------------- |
| `redirectUris` | array | Yes      | Allowed redirect URIs           |
| `scopes`       | array | Yes      | Allowed OAuth scopes            |

### Update OAuth Settings

```
PATCH /v1/applications/{botId}/oauth
```

Updates redirect URIs and/or scopes.

### Reset Client Secret

```
POST /v1/applications/{botId}/oauth/reset-secret
```

Regenerates the OAuth client secret and revokes all existing tokens.
