# Users

## User Object

```json
{
	"id": "user_abc123",
	"username": "alice",
	"global_name": "Alice",
	"avatar": "https://cdn.example.com/avatars/abc.png",
	"banner": "https://cdn.example.com/banners/abc.png",
	"accent_color": 3447003,
	"bio": "Building cool things",
	"pronouns": "she/her",
	"bot": false,
	"system": false,
	"flags": 0,
	"public_flags": 0
}
```

| Field          | Type           | Description                         |
| -------------- | -------------- | ----------------------------------- |
| `id`           | string         | The user's unique ID                |
| `username`     | string         | Unique username                     |
| `global_name`  | string \| null | Display name shown in the UI        |
| `avatar`       | string \| null | Avatar image URL                    |
| `banner`       | string \| null | Profile banner image URL            |
| `accent_color` | number \| null | Profile accent color as an integer  |
| `bio`          | string \| null | User bio / about me (max 190 chars) |
| `pronouns`     | string \| null | Preferred pronouns (max 40 chars)   |
| `bot`          | boolean        | Whether this is a bot account       |
| `system`       | boolean        | Whether this is a system account    |
| `flags`        | number         | User flags bitfield                 |
| `public_flags` | number         | Public user flags bitfield          |

## Endpoints

### Get Current User

```
GET /v1/users/@me
```

Returns the authenticated user's profile.

**Response 200**: [User object](#user-object).

---

### Update Current User

```
PATCH /v1/users/@me
```

Updates the current user's profile.

**Request body** — all fields optional:

| Field          | Type           | Constraints   | Description    |
| -------------- | -------------- | ------------- | -------------- |
| `username`     | string         | —             | Username       |
| `display_name` | string \| null | Max 32 chars  | Display name   |
| `avatar`       | string \| null | URL           | Avatar URL     |
| `banner`       | string \| null | URL           | Banner URL     |
| `accent_color` | number \| null | Integer       | Accent color   |
| `bio`          | string \| null | Max 190 chars | Bio / about me |
| `pronouns`     | string \| null | Max 40 chars  | Pronouns       |

Set any field to `null` to clear it.

**Response 200**: Updated [user object](#user-object).

---

### Get User Profile

```
GET /v1/users/{userId}
```

Returns the public profile of any user.

**Response 200**: [User object](#user-object).

## Presence

### Get User Presence

```
GET /v1/users/{userId}/presence
```

Returns the current presence status and activity for any user. Activities may be hidden based on privacy settings.

**Response 200**:

```json
{
	"user_id": "user_abc123",
	"status": "online",
	"custom_status": "Writing code",
	"last_seen": "2024-01-15T14:30:00.000Z",
	"activities": {}
}
```

| `status` value | Description                               |
| -------------- | ----------------------------------------- |
| `online`       | User is active                            |
| `idle`         | User is idle / AFK                        |
| `dnd`          | Do Not Disturb — suppresses notifications |
| `offline`      | User is offline or invisible              |

---

### Update Own Presence

```
PUT /v1/users/@me/presence
```

Updates your presence status. A `PRESENCE_UPDATE` gateway event is broadcast to every guild the user belongs to.

**Request body**:

| Field          | Type   | Required | Description                               |
| -------------- | ------ | -------- | ----------------------------------------- |
| `status`       | string | Yes      | One of `online`, `idle`, `dnd`, `offline` |
| `customStatus` | string | No       | Free-text status message                  |
| `activities`   | array  | No       | List of activity objects                  |

---

### Set Activities

```
PUT /v1/users/@me/presence/activity
```

Sets the user's current activities (e.g., playing a game, listening to music). External apps should use OAuth with the `activities.write` scope.

**Request body**:

| Field        | Type  | Required | Description              |
| ------------ | ----- | -------- | ------------------------ |
| `activities` | array | Yes      | List of activity objects |

## Read States

### Get Read States

```
GET /v1/users/@me/read-state
```

Returns the last-read position and unread mention count for every channel the user has interacted with.

**Response 200**:

```json
[
	{
		"channel_id": "ch_general",
		"last_message_id": "msg_999",
		"last_read_at": "2024-01-15T10:00:00.000Z",
		"mention_count": 2
	}
]
```

---

### Acknowledge Messages

```
POST /v1/channels/{channelId}/ack/{messageId}
```

Marks all messages up to `messageId` as read in the channel. Resets the mention count to zero.

**Response 200**:

```json
{
	"channel_id": "ch_general",
	"message_id": "msg_999"
}
```

---

## Sessions

### List Sessions

```
GET /v1/users/@me/sessions
```

Returns all active sessions for the current user.

**Response 200**:

```json
[
	{
		"id": "sess_abc",
		"user_agent": "Mozilla/5.0...",
		"ip_address": "192.168.1.1",
		"last_active_at": "2024-01-15T10:00:00.000Z",
		"created_at": "2024-01-01T00:00:00.000Z",
		"expires_at": "2024-02-01T00:00:00.000Z",
		"is_current": true
	}
]
```

| Field            | Type           | Description                                |
| ---------------- | -------------- | ------------------------------------------ |
| `id`             | string         | Session ID                                 |
| `user_agent`     | string \| null | Browser/client user agent string           |
| `ip_address`     | string \| null | IP address of the session                  |
| `last_active_at` | string         | ISO 8601 timestamp of last activity        |
| `created_at`     | string         | ISO 8601 timestamp when session was created |
| `expires_at`     | string         | ISO 8601 timestamp when session expires    |
| `is_current`     | boolean        | Whether this is the current active session |

---

### Revoke Session

```
DELETE /v1/users/@me/sessions/{sessionId}
```

Revokes a specific session by ID, signing out that device/client.

**Response 200**:

```json
{
	"id": "sess_abc"
}
```

## Direct Messages

### Open DM Channel

```
POST /v1/users/@me/channels
```

Opens a DM channel with another user. If a DM channel already exists, returns the existing one.

**Request body**:

| Field          | Type   | Required | Description             |
| -------------- | ------ | -------- | ----------------------- |
| `recipient_id` | string | Yes      | The recipient's user ID |

**Response 200**:

```json
{
	"id": "dm_abc",
	"type": "dm",
	"recipients": [
		{
			"id": "user_def456",
			"username": "bob",
			"global_name": "Bob",
			"avatar": null
		}
	],
	"created_at": "2024-01-15T14:30:00.000Z"
}
```

---

### List DM Channels

```
GET /v1/users/@me/channels
```

Returns all open DM channels, ordered by most recently active, with last message preview.

**Response 200** — array of:

```json
{
	"id": "dm_abc",
	"type": "dm",
	"recipients": [...],
	"last_message": {},
	"created_at": "2024-01-15T14:30:00.000Z"
}
```

---

### Get DM Messages

```
GET /v1/channels/{channelId}/dm-messages
```

Fetches messages in a DM channel. Defaults to 50 messages, max 100. Supports cursor-based pagination.

| Parameter | Description                          |
| --------- | ------------------------------------ |
| `limit`   | Max messages to return (default 50)  |
| `before`  | ISO timestamp cursor (messages before) |
| `after`   | ISO timestamp cursor (messages after)  |

**Response 200** — array of:

```json
{
	"id": "msg_abc",
	"channel_id": "dm_abc",
	"content": "Hey!",
	"sender": {},
	"created_at": "2024-01-15T14:30:00.000Z",
	"edited_at": null
}
```

---

### Send DM Message

```
POST /v1/channels/{channelId}/dm-messages
```

**Request body**:

| Field     | Type   | Required | Description      |
| --------- | ------ | -------- | ---------------- |
| `content` | string | Yes      | Message content  |

---

### Leave DM Channel

```
DELETE /v1/channels/{channelId}/dm
```

Closes or leaves a DM channel by removing the current user as a participant.
