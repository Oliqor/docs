# Channels & Messages

## Channel Types

| Type            | Numeric | Description                        |
| --------------- | ------- | ---------------------------------- |
| Text            | `0`     | Standard text channel              |
| DM              | `1`     | Direct message (one-on-one)        |
| Voice           | `2`     | Real-time audio/video              |
| Group DM        | `3`     | Group direct message               |
| Category        | `4`     | Organizational grouping            |
| Announcement    | `5`     | Announcement channel               |
| Public Thread   | `11`    | Public thread under a text channel |
| Private Thread  | `12`    | Invite-only thread                 |
| Stage           | `13`    | Stage voice channel                |
| Guild Directory | `14`    | Server directory listing           |
| Forum           | `15`    | Discussion-oriented channel        |
| Media           | `16`    | Media-focused channel              |

## Channel Object

```json
{
	"id": "ch_general",
	"type": 0,
	"guild_id": "guild_abc",
	"name": "general",
	"topic": "General discussion",
	"position": 0,
	"parent_id": "ch_category_1",
	"nsfw": false,
	"rate_limit_per_user": 0,
	"last_message_id": null,
	"permission_overwrites": [
		{
			"id": "role_everyone",
			"type": 0,
			"allow": "0",
			"deny": "256"
		}
	]
}
```

Voice channels include additional fields:

```json
{
	"bitrate": 64000,
	"user_limit": 10,
	"rtc_region": "us-east",
	"video_quality_mode": 1
}
```

Thread channels include:

```json
{
	"flags": 0
}
```

## Channel Endpoints

### Create Channel

```
POST /v1/guilds/{guildId}/channels
```

Requires `MANAGE_CHANNELS` permission.

**Request body**:

| Field          | Type   | Required | Description                      |
| -------------- | ------ | -------- | -------------------------------- |
| `name`         | string | Yes      | Channel name                     |
| `topic`        | string | No       | Channel topic                    |
| `type`         | number | No       | Channel type (default: `0`)      |
| `parentId`     | string | No       | Category channel ID              |
| `bitrate`      | number | No       | Audio bitrate (voice channels)   |
| `userLimit`    | number | No       | Max users (voice, 0 = unlimited) |
| `rtcRegion`    | string | No       | Voice region override            |
| `videoQuality` | string | No       | `"auto"` or `"full"`             |

**Response 200**: Channel object.

Fires a `CHANNEL_CREATE` gateway event. Creates an audit log entry.

---

### List Channels

```
GET /v1/guilds/{guildId}/channels
```

Returns all channels ordered by position.

---

### Update Channel

```
PATCH /v1/channels/{channelId}
```

Requires `MANAGE_CHANNELS` permission. All fields optional.

**Request body**:

| Field      | Type   | Description    |
| ---------- | ------ | -------------- |
| `name`     | string | Channel name   |
| `topic`    | string | Channel topic  |
| `position` | number | Channel position |

Fires a `CHANNEL_UPDATE` gateway event.

---

### Delete Channel

```
DELETE /v1/channels/{channelId}
```

Requires `MANAGE_CHANNELS` permission.

Fires a `CHANNEL_DELETE` gateway event.

---

### Trigger Typing Indicator

```
POST /v1/channels/{channelId}/typing
```

Broadcasts a `TYPING_START` gateway event to all clients watching the channel.

## Threads

Threads are lightweight sub-channels that live under a text channel.

### Create Thread

```
POST /v1/channels/{channelId}/threads
```

**Request body**:

| Field       | Type   | Required | Description                                |
| ----------- | ------ | -------- | ------------------------------------------ |
| `name`      | string | Yes      | Thread name                                |
| `messageId` | string | No       | Parent message to start the thread from    |
| `type`      | number | No       | `11` = public thread, `12` = private thread |

**Response 200**: Channel object.

Fires `THREAD_CREATE` and `THREAD_MEMBER_UPDATE` gateway events.

---

### List Threads

```
GET /v1/channels/{channelId}/threads
```

**Query parameters**:

| Parameter  | Type    | Description                                                   |
| ---------- | ------- | ------------------------------------------------------------- |
| `archived` | boolean | If `true`, return archived threads only                       |
| `before`   | string  | ISO 8601 timestamp — return threads archived before this time |
| `limit`    | number  | Max threads to return (default 50)                            |

**Response 200**:

```json
{
	"threads": [...],
	"has_more": false
}
```

---

### List Active Threads

```
GET /v1/guilds/{guildId}/threads/active
```

Returns all non-archived threads across all channels in the guild.

---

### Update Thread

```
PATCH /v1/threads/{threadId}
```

---

### Delete Thread

```
DELETE /v1/threads/{threadId}
```

Requires `MANAGE_CHANNELS` permission.

---

### Thread Members

| Endpoint                                               | Method | Description                 |
| ------------------------------------------------------ | ------ | --------------------------- |
| `/v1/channels/{channelId}/thread-members/@me`          | PUT    | Join a thread               |
| `/v1/channels/{channelId}/thread-members/@me`          | DELETE | Leave a thread              |
| `/v1/channels/{channelId}/thread-members/{userId}`     | PUT    | Add a user to a thread      |
| `/v1/channels/{channelId}/thread-members/{userId}`     | DELETE | Remove a user from a thread |
| `/v1/channels/{channelId}/thread-members`              | GET    | List all thread members     |

## Messages

### Message Object

```json
{
	"id": "msg_123",
	"channel_id": "ch_general",
	"content": "Hello, world!",
	"author": {
		"id": "user_abc123",
		"username": "alice",
		"global_name": "Alice",
		"avatar": "https://cdn.example.com/avatars/abc.png"
	},
	"created_at": "2024-01-15T14:30:00.000Z",
	"edited_timestamp": null,
	"nonce": "client-dedup-123",
	"flags": 0,
	"reactions": [],
	"reference": null
}
```

---

### Send Message

```
POST /v1/channels/{channelId}/messages
```

**Request body**:

| Field                  | Type   | Required | Description                             |
| ---------------------- | ------ | -------- | --------------------------------------- |
| `content`              | string | Yes      | Message content                         |
| `reference_message_id` | string | No       | Reply to this message                   |
| `nonce`                | string | No       | Client-side deduplication token         |

Fires a `MESSAGE_CREATE` gateway event.

---

### Get Messages

```
GET /v1/channels/{channelId}/messages
```

Cursor-based pagination. Use `before`, `after`, or `around`.

| Parameter | Type   | Default | Description                |
| --------- | ------ | ------- | -------------------------- |
| `limit`   | number | 50      | Messages to return (1–100) |
| `before`  | string | —       | Messages before this ID    |
| `after`   | string | —       | Messages after this ID     |
| `around`  | string | —       | Messages around this ID    |

---

### Get Single Message

```
GET /v1/channels/{channelId}/messages/{messageId}
```

---

### Edit Message

```
PATCH /v1/channels/{channelId}/messages/{messageId}
```

Only the original author can edit their message.

**Request body**:

| Field     | Type   | Required | Description     |
| --------- | ------ | -------- | --------------- |
| `content` | string | Yes      | Updated content |

Fires a `MESSAGE_UPDATE` gateway event.

---

### Delete Message

```
DELETE /v1/channels/{channelId}/messages/{messageId}
```

The original author can always delete their own messages. Others need `MANAGE_MESSAGES`.

Fires a `MESSAGE_DELETE` gateway event.

---

### Bulk Delete Messages

```
POST /v1/channels/{channelId}/messages/bulk-delete
```

Requires `MANAGE_MESSAGES` permission. Deletes 1–100 messages at once.

**Request body**:

| Field        | Type     | Required | Description              |
| ------------ | -------- | -------- | ------------------------ |
| `message_ids`| string[] | Yes      | Array of message IDs     |

**Response 200**:

```json
{
	"deleted": ["msg_1", "msg_2"],
	"failed": ["msg_3"]
}
```

---

### Crosspost Message

```
POST /v1/channels/{channelId}/messages/{messageId}/crosspost
```

Publishes a message from an announcement channel. Requires `SEND_MESSAGES` or `MANAGE_MESSAGES`.

---

### Search Channel Messages

```
GET /v1/channels/{channelId}/messages/search
```

Full-text search within a channel. Requires `READ_MESSAGE_HISTORY` permission.

| Parameter  | Type   | Description                      |
| ---------- | ------ | -------------------------------- |
| `q`        | string | Search query                     |
| `authorId` | string | Filter by author user ID         |
| `before`   | string | Messages before this message ID  |
| `after`    | string | Messages after this message ID   |
| `limit`    | number | Max results                      |

## Reactions

### Add Reaction

```
PUT /v1/channels/{channelId}/messages/{messageId}/reactions/{emoji}
```

`emoji` is URL-encoded (e.g., `%F0%9F%91%8D` for 👍). Idempotent.

**Response 200**:

```json
{
	"message_id": "msg_123",
	"emoji": "👍",
	"count": 3
}
```

---

### Remove Reaction

```
DELETE /v1/channels/{channelId}/messages/{messageId}/reactions/{emoji}
```

Removes your own reaction.

---

### Remove All Reactions for Emoji

```
DELETE /v1/channels/{channelId}/messages/{messageId}/reactions/{emoji}/all
```

Requires `MANAGE_MESSAGES`.

---

### List Reactions

```
GET /v1/channels/{channelId}/messages/{messageId}/reactions
```

Returns all reactions grouped by emoji with reacting user info.

**Response 200** — array of:

```json
{
	"emoji": "👍",
	"emoji_id": null,
	"count": 3,
	"users": []
}
```

---

## Webhooks

Webhooks allow external services to send messages to a channel without a bot account.

### Webhook Object

```json
{
	"id": "wh_abc",
	"channelId": "ch_general",
	"communityId": "guild_abc",
	"name": "GitHub Notifications",
	"avatarUrl": null,
	"token": "webhook-secret-token",
	"createdAt": "2024-01-15T14:30:00.000Z"
}
```

---

### List Channel Webhooks

```
GET /v1/channels/{channelId}/webhooks
```

Requires `MANAGE_CHANNELS` permission.

---

### Create Webhook

```
POST /v1/channels/{channelId}/webhooks
```

Requires `MANAGE_CHANNELS` permission. The response includes the `token` — store it securely.

**Request body**:

| Field       | Type   | Required | Description       |
| ----------- | ------ | -------- | ----------------- |
| `name`      | string | Yes      | Webhook name      |
| `avatarUrl` | string | No       | Webhook avatar URL |

---

### Get Webhook

```
GET /v1/webhooks/{webhookId}
```

### Update Webhook

```
PATCH /v1/webhooks/{webhookId}
```

**Request body** (all optional): `name`, `avatarUrl`, `channelId`

### Delete Webhook

```
DELETE /v1/webhooks/{webhookId}
```

Requires `MANAGE_CHANNELS` permission.

---

### Get Webhook by Token

```
GET /v1/webhooks/{webhookId}/{webhookToken}
```

No authentication required.

---

### Execute Webhook

```
POST /v1/webhooks/{webhookId}/{webhookToken}
```

No authentication required — the token in the URL serves as the credential.

**Request body**:

| Field        | Type   | Description       |
| ------------ | ------ | ----------------- |
| `content`    | string | Message content   |
| `embeds`     | array  | Embed objects     |
| `components` | array  | Component objects |
| `flags`      | number | Message flags     |

### Edit Original Interaction Response

```
PATCH /v1/webhooks/{webhookId}/{webhookToken}/messages/@original
```

### Delete Original Interaction Response

```
DELETE /v1/webhooks/{webhookId}/{webhookToken}/messages/@original
```

## Permission Overrides

### Get Channel Permission Overrides

```
GET /v1/guilds/{guildId}/channels/{channelId}/permissions
```

**Response 200** — array of:

```json
{
	"id": "role_mod",
	"type": 0,
	"allow": "104324673",
	"deny": "0"
}
```

| `type` | Target  |
| ------ | ------- |
| `0`    | Role    |
| `1`    | Member  |

---

### Set Channel Permission Override

```
PUT /v1/guilds/{guildId}/channels/{channelId}/permissions/{overrideId}
```

Requires `MANAGE_CHANNELS`.

**Request body**:

| Field   | Type   | Required | Description                  |
| ------- | ------ | -------- | ---------------------------- |
| `type`  | number | Yes      | `0` for role, `1` for member |
| `allow` | string | No       | Allowed permission bit set   |
| `deny`  | string | No       | Denied permission bit set    |

---

### Delete Channel Permission Override

```
DELETE /v1/guilds/{guildId}/channels/{channelId}/permissions/{overrideId}
```

Requires `MANAGE_CHANNELS`.
