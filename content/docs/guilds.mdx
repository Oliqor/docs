# Guilds

A **guild** is the top-level grouping in Oliqor (equivalent to a "server" in Discord). Guilds contain channels, members, roles, emojis, stickers, and more.

## Guild Object

```json
{
	"id": "guild_abc",
	"name": "My Guild",
	"icon": "https://cdn.example.com/icons/abc.png",
	"description": "A place to hang out",
	"owner_id": "user_abc123",
	"system_channel_id": "ch_general",
	"system_channel_flags": 0,
	"verification_level": 0,
	"default_message_notifications": 0,
	"explicit_content_filter": 0,
	"mfa_level": 0,
	"premium_tier": 0,
	"premium_subscription_count": 0,
	"nsfw_level": 0,
	"preferred_locale": "en-US",
	"afk_channel_id": null,
	"afk_timeout": 300,
	"max_members": 500000,
	"roles": []
}
```

| Field                            | Type           | Description                                            |
| -------------------------------- | -------------- | ------------------------------------------------------ |
| `id`                             | string         | Guild ID                                               |
| `name`                           | string         | Guild name                                             |
| `icon`                           | string \| null | Icon image URL                                         |
| `description`                    | string \| null | Guild description                                      |
| `owner_id`                       | string         | User ID of the guild owner                             |
| `afk_channel_id`                 | string \| null | Voice channel ID for AFK members                       |
| `afk_timeout`                    | number         | Seconds before moving to AFK channel                   |
| `verification_level`             | number         | Verification level (0–4)                               |
| `default_message_notifications`  | number         | Default notification setting (0 = all, 1 = mentions)  |
| `explicit_content_filter`        | number         | Explicit content filter level (0–2)                    |
| `mfa_level`                      | number         | MFA requirement for moderation (0 = off, 1 = required) |
| `system_channel_id`              | string \| null | Channel for system messages                            |
| `system_channel_flags`           | number         | System channel flags bitfield                          |
| `preferred_locale`               | string         | Preferred locale (e.g., `"en-US"`)                     |
| `premium_tier`                   | number         | Boost tier (0–3)                                       |
| `premium_subscription_count`     | number         | Number of active boosts                                |
| `nsfw_level`                     | number         | NSFW level (0–3)                                       |
| `max_members`                    | number         | Maximum member count                                   |
| `roles`                          | Role[]         | List of guild roles                                    |

## Guild Endpoints

### Create Guild

```
POST /v1/guilds
```

Creates a new guild. A `#general` text channel and an `@everyone` role with [default permissions](./permissions#default-member-permissions) are created automatically. The creator becomes the owner.

**Request body**:

| Field         | Type   | Required | Constraints   |
| ------------- | ------ | -------- | ------------- |
| `name`        | string | Yes      | 1–100 chars   |
| `description` | string | No       | Max 500 chars |
| `icon`        | string | No       | URL           |

**Response 200**: Full [guild object](#guild-object).

---

### Get Guild

```
GET /v1/guilds/{guildId}
```

Returns the full guild object including roles.

---

### Update Guild

```
PATCH /v1/guilds/{guildId}
```

Requires `MANAGE_COMMUNITY` permission. All fields optional.

**Request body**:

| Field         | Type   | Description         |
| ------------- | ------ | ------------------- |
| `name`        | string | Guild name          |
| `description` | string | Guild description   |
| `icon`        | string | Icon URL            |

Fires a `GUILD_UPDATE` gateway event. Creates an audit log entry.

---

### Delete Guild

```
DELETE /v1/guilds/{guildId}
```

Owner only. Permanently deletes the guild and all associated data. This action is irreversible.

Fires a `GUILD_DELETE` gateway event.

---

### Get Vanity URL

```
GET /v1/guilds/{guildId}/vanity-url
```

Returns the guild's vanity URL code and current use count.

**Response 200**:

```json
{
	"code": "my-guild",
	"uses": 42
}
```

---

### Set Vanity URL

```
PATCH /v1/guilds/{guildId}/vanity-url
```

Requires `MANAGE_COMMUNITY` permission.

**Request body**:

| Field  | Type   | Required | Description         |
| ------ | ------ | -------- | ------------------- |
| `code` | string | Yes      | New vanity URL code |

---

### Get Widget Settings

```
GET /v1/guilds/{guildId}/widget
```

Requires `MANAGE_COMMUNITY` permission.

**Response 200**:

```json
{
	"enabled": false,
	"channelId": null
}
```

---

### Update Widget Settings

```
PATCH /v1/guilds/{guildId}/widget
```

Requires `MANAGE_COMMUNITY` permission.

**Request body**:

| Field       | Type    | Description                          |
| ----------- | ------- | ------------------------------------ |
| `enabled`   | boolean | Enable or disable the widget         |
| `channelId` | string  | Channel to invite to from the widget |

---

### Prune Members

```
GET /v1/guilds/{guildId}/prune
```

Preview how many members would be pruned. Requires `KICK_MEMBERS` permission.

**Query parameters**:

| Parameter      | Type   | Description                                        |
| -------------- | ------ | -------------------------------------------------- |
| `days`         | number | Days of inactivity threshold                       |
| `includeRoles` | string | Comma-separated role IDs to include in prune count |

**Response 200**: `{ "pruned": 12 }`

```
POST /v1/guilds/{guildId}/prune
```

Execute member pruning. Requires `KICK_MEMBERS` permission.

**Request body**:

| Field               | Type    | Description                                  |
| ------------------- | ------- | -------------------------------------------- |
| `days`              | number  | Days of inactivity (default: 7)              |
| `computePruneCount` | boolean | Whether to return pruned count (can be slow) |
| `reason`            | string  | Audit log reason                             |

**Response 200**: `{ "pruned": 12 }` (or `null` if `computePruneCount` is false)

## Members

### Member Object

```json
{
	"user": {
		"id": "user_abc123",
		"username": "alice",
		"global_name": "Alice",
		"avatar": "https://cdn.example.com/avatars/abc.png"
	},
	"nick": "ali",
	"avatar": null,
	"roles": ["role_mod", "role_vip"],
	"joined_at": "2024-01-01T00:00:00.000Z",
	"flags": 0,
	"pending": false,
	"member_id": "member_abc"
}
```

| Field       | Type           | Description                                           |
| ----------- | -------------- | ----------------------------------------------------- |
| `user`      | User object    | The underlying user (may be omitted in some contexts) |
| `nick`      | string \| null | Guild-specific nickname                               |
| `avatar`    | string \| null | Guild-specific avatar URL                             |
| `roles`     | string[]       | Array of role IDs the member has                      |
| `joined_at` | string         | ISO 8601 timestamp of when they joined                |
| `flags`     | number         | Member flags bitfield                                 |
| `pending`   | boolean        | Whether the member has passed membership screening    |
| `member_id` | string         | Membership record ID                                  |

---

### List Guild Members

```
GET /v1/guilds/{guildId}/members
```

**Query parameters**:

| Parameter | Type   | Default | Description                       |
| --------- | ------ | ------- | --------------------------------- |
| `limit`   | number | 100     | Max members (1–1000)              |
| `after`   | string | —       | Return members after this user ID |

**Response 200**: Array of [member objects](#member-object).

---

### Search Members

```
GET /v1/guilds/{guildId}/members/search
```

Search for members by username or nickname.

**Query parameters**:

| Parameter | Type   | Required | Description               |
| --------- | ------ | -------- | ------------------------- |
| `query`   | string | Yes      | Search query              |
| `limit`   | number | No       | Max results               |

---

### Get Own Membership

```
GET /v1/guilds/{guildId}/members/@me
```

Returns the authenticated user's member object in the guild.

---

### Get My Permissions

```
GET /v1/guilds/{guildId}/members/@me/permissions
```

Returns the computed permission bitfield for the authenticated user in the guild.

**Response 200**:

```json
{
	"permissions": "104324673",
	"isAdministrator": false
}
```

---

### Update Own Member Profile

```
PATCH /v1/guilds/{guildId}/members/@me
```

Update your own nickname and per-guild avatar.

**Request body** — all fields optional:

| Field    | Type           | Description           |
| -------- | -------------- | --------------------- |
| `nick`   | string \| null | Guild nickname        |
| `avatar` | string \| null | Guild-specific avatar |

Set to `null` to clear. Fires a `GUILD_MEMBER_UPDATE` gateway event.

---

### Join Guild

```
POST /v1/guilds/{guildId}/members
```

Join a public guild directly (without invite).

---

### Leave Guild

```
DELETE /v1/guilds/{guildId}/members/@me
```

Leaves the guild. You cannot leave a guild you own — delete it instead.

---

### Get Guild Member

```
GET /v1/guilds/{guildId}/members/{memberId}
```

Returns a single member by their membership ID.

---

### Update Guild Member

```
PATCH /v1/guilds/{guildId}/members/{memberId}
```

**Request body** — all fields optional:

| Field      | Type           | Description              |
| ---------- | -------------- | ------------------------ |
| `nickname` | string \| null | Set the member's nickname |
| `roles`    | string[]       | Set the member's roles   |

---

### Kick Member

```
DELETE /v1/guilds/{guildId}/members/{memberId}
```

Requires `KICK_MEMBERS` permission. Fires a `GUILD_MEMBER_REMOVE` gateway event. Creates an audit log entry.

---

### Add Role to Member

```
PUT /v1/guilds/{guildId}/members/{memberId}/roles/{roleId}
```

Requires `MANAGE_ROLES` permission.

**Response 200**:

```json
{
	"memberId": "member_abc",
	"roleId": "role_mod",
	"roles": ["role_mod", "role_vip"]
}
```

---

### Remove Role from Member

```
DELETE /v1/guilds/{guildId}/members/{memberId}/roles/{roleId}
```

Requires `MANAGE_ROLES` permission.

## Roles

### Role Object

```json
{
	"id": "role_mod",
	"name": "Moderator",
	"color": 3447003,
	"hoist": true,
	"position": 2,
	"permissions": "104324673",
	"managed": false,
	"mentionable": true
}
```

| Field         | Type    | Description                                                           |
| ------------- | ------- | --------------------------------------------------------------------- |
| `id`          | string  | Role ID                                                               |
| `name`        | string  | Role name                                                             |
| `color`       | number  | Integer color value (0 = no color)                                    |
| `hoist`       | boolean | Whether this role is displayed separately in the member sidebar       |
| `position`    | number  | Position in the role hierarchy (higher = more powerful)               |
| `permissions` | string  | Permission bitfield as a string (see [Permissions](./permissions))    |
| `managed`     | boolean | Whether this role is managed by an integration                        |
| `mentionable` | boolean | Whether this role can be mentioned by anyone                          |

---

### List Guild Roles

```
GET /v1/guilds/{guildId}/roles
```

Returns all roles ordered by position.

---

### Create Role

```
POST /v1/guilds/{guildId}/roles
```

Requires `MANAGE_ROLES` permission.

**Request body**:

| Field         | Type    | Required | Description                |
| ------------- | ------- | -------- | -------------------------- |
| `name`        | string  | Yes      | Role name                  |
| `color`       | number  | No       | Color integer              |
| `permissions` | string  | No       | Permission bitfield string |
| `position`    | number  | No       | Position in hierarchy      |
| `mentionable` | boolean | No       | Whether mentionable        |
| `hoisted`     | boolean | No       | Whether displayed separately |

Fires a `GUILD_ROLE_CREATE` gateway event.

---

### Bulk Update Role Positions

```
PATCH /v1/guilds/{guildId}/roles
```

Requires `MANAGE_ROLES` permission. Accepts an array of `{ id, position }` objects.

---

### Update Role

```
PATCH /v1/guilds/{guildId}/roles/{roleId}
```

Requires `MANAGE_ROLES` permission. All fields optional.

| Field         | Type    | Description                |
| ------------- | ------- | -------------------------- |

| `permissions` | string  | Permission bitfield string |
| `position`    | number  | Position in hierarchy      |

### Delete Role

```
DELETE /v1/guilds/{guildId}/roles/{roleId}
```

## Invites

### Create Invite

```
POST /v1/guilds/{guildId}/invites
```

Requires `CREATE_INVITES` permission.

**Request body** — all fields optional:

| Field            | Type   | Default         | Description                      |
| ---------------- | ------ | --------------- | -------------------------------- |
| `maxUses`        | number | `0` (unlimited) | Max times the invite can be used |
| `expiresInHours` | number | `0` (never)     | Hours until the invite expires   |

**Response 200**:

```json
{
	"code": "abc123",
	"guild": {},
	"uses": 0,
	"max_uses": 10,
	"max_age": 86400,
	"temporary": false,
	"created_at": "2024-01-15T14:30:00.000Z",
	"expires_at": "2024-01-16T14:30:00.000Z"
}
```

---

### Get Invite Info

```
GET /v1/invites/{code}
```

No authentication required. Returns guild info for the invite without joining. Returns 404 if expired or exhausted.

---

### Join via Invite

```
POST /v1/invites/{code}
```

Consumes an invite code and joins the guild.

**Response 200**:

```json
{
	"guild_id": "guild_abc",
	"guild_name": "My Guild",
	"user_id": "user_abc123",
	"joined_at": "2024-01-15T14:30:00.000Z"
}
```

---

### List Guild Invites

```
GET /v1/guilds/{guildId}/invites
```

Requires `MANAGE_INVITES` permission.

**Query parameters**: `limit`, `after`

---

### Delete Invite

```
DELETE /v1/guilds/{guildId}/invites/{code}
```

The invite creator or anyone with `MANAGE_INVITES` can delete.

## Scheduled Events

### List Scheduled Events

```
GET /v1/guilds/{guildId}/scheduled-events
```

**Query parameters**:

| Parameter       | Type    | Description                   |
| --------------- | ------- | ----------------------------- |
| `withUserCount` | boolean | Include interested user count |
| `limit`         | number  | Max events to return          |
| `after`         | string  | Return events after this ID   |

### Get Scheduled Event

```
GET /v1/guilds/{guildId}/scheduled-events/{eventId}
```

### Create Scheduled Event

```
POST /v1/guilds/{guildId}/scheduled-events
```

Requires `MANAGE_COMMUNITY` permission.

**Request body**:

| Field            | Type   | Required | Description                                      |
| ---------------- | ------ | -------- | ------------------------------------------------ |
| `name`           | string | Yes      | Event name                                       |
| `entityType`     | string | Yes      | `STAGE_INSTANCE`, `VOICE`, or `EXTERNAL`         |
| `scheduledStart` | string | Yes      | ISO 8601 start time                              |
| `description`    | string | No       | Event description                                |
| `channelId`      | string | No       | Channel for the event (required for voice/stage) |
| `scheduledEnd`   | string | No       | ISO 8601 end time                                |
| `privacyLevel`   | string | No       | Privacy level                                    |
| `entityMetadata` | object | No       | Extra metadata (e.g., `location` for EXTERNAL)   |
| `imageUrl`       | string | No       | Event cover image URL                            |

**Response 200** includes: `id`, `communityId`, `channelId`, `name`, `description`, `scheduledStart`, `scheduledEnd`, `privacyLevel`, `status`, `entityType`, `entityMetadata`, `imageUrl`, `creatorId`, `createdAt`.

### Update Scheduled Event

```
PATCH /v1/guilds/{guildId}/scheduled-events/{eventId}
```

Requires `MANAGE_COMMUNITY` permission. All fields optional. Also accepts `status`:

| `status`    | Description                |
| ----------- | -------------------------- |
| `SCHEDULED` | Event is scheduled         |
| `ACTIVE`    | Event is currently running |
| `COMPLETED` | Event has ended            |
| `CANCELLED` | Event was cancelled        |

### Delete Scheduled Event

```
DELETE /v1/guilds/{guildId}/scheduled-events/{eventId}
```

Requires `MANAGE_COMMUNITY` permission.

### Express Interest

```
PUT /v1/guilds/{guildId}/scheduled-events/{eventId}/interest
```

### Remove Interest

```
DELETE /v1/guilds/{guildId}/scheduled-events/{eventId}/interest
```

## Message Search

### Search Channel Messages

```
GET /v1/channels/{channelId}/messages/search
```

Full-text search within a channel. Requires `READ_MESSAGE_HISTORY` permission.

**Query parameters**:

| Parameter  | Type   | Description                     |
| ---------- | ------ | ------------------------------- |
| `q`        | string | Search query                    |
| `authorId` | string | Filter by author user ID        |
| `before`   | string | Messages before this message ID |
| `after`    | string | Messages after this message ID  |
| `limit`    | number | Max results                     |

### Search Guild Messages

```
GET /v1/guilds/{guildId}/messages/search
```

Full-text search across all channels in a guild. Requires `READ_MESSAGE_HISTORY` permission.

**Query parameters**:

| Parameter   | Type   | Description                |
| ----------- | ------ | -------------------------- |
| `q`         | string | Search query               |
| `channelId` | string | Filter to specific channel |
| `authorId`  | string | Filter by author user ID   |
| `limit`     | number | Max results                |
